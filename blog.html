Global Scope vs Block Scope

# Global scope Block Scope
1 The variables defined outside of any function or curly brackets are known as global variables and have global scope. The block scope can be defined as the scope of the variables inside the curly brackets {}
2 Global scope means that the variables can be accessed from any part of that program You are only allowed to refer to these variables from within the curly brackets {}.
3 The keywords var is used to define block variables. The keywords let and const are used to define block variables.

How jvascript event loop works

How jvascript event loop works
The Event Loop has one simple job — to monitor the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and will push it to the Call Stack, which effectively runs it. Such an iteration is called a tick in the Event Loop.
Also we can say The event loop is a process that waits for the Call Stack to be clear before pushing callbacks from the Task Queue to the Call Stack. Once the Stack is clear, the event loop triggers and checks the Task Queue for available callbacks. If there are any, it pushes it to the Call Stack, waits for the Call Stack to be clear again, and repeats the same process.

How many ways to get Undefined

How many ways to get Undefined :
Variable that is not initialized will give undefined.
Function with no return.
Parameter that is not passed in function will be undefined.
If return has nothing on the right side will return undefined.
Property that doesn’t exist on an object will give you undefined.
Accessing array elements outside of the index range.
Deleting an element inside an array.
Set a value directly to undefined.
